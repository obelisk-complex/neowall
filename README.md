## What is this?

NeoWall renders **Shadertoy shaders** as your live desktop wallpaper. C, EGL, OpenGL 3.3, multi-monitor, 60 fps, low CPU. Wayland and X11.

```bash
neowall
```

This fork (`obelisk-complex/neowall`) layers cross-WM compatibility, multi-monitor X11 correctness, performance work, and a few new configuration knobs on top of upstream `1ay1/neowall`.

## What's new in this fork

See [`CHANGELOG.md`](CHANGELOG.md) for the full list. Headlines:

- **X11 multi-monitor actually works.** Wallpaper covers the bounding box of every active RandR monitor, not just the first CRTC. Place at the virtual-screen origin so monitors at non-(0,0) coordinates are covered.
- **Cinnamon, KDE, GNOME-X11, Xfce, Compiz, Marco, Openbox, Fluxbox, Awesome are first-class.** EWMH window-manager detection chooses a managed `_NET_WM_WINDOW_TYPE_DESKTOP` window on compositing managers, falls back to override-redirect on tiling WMs (i3, bspwm, dwm, xmonad). Override with `NEOWALL_X11_OVERRIDE_REDIRECT=0|1`.
- **KDE Plasma 6 / KWin 6 fix.** Layer surface uses `LAYER_BACKGROUND` with `exclusive_zone=0` — the previous `LAYER_BOTTOM` + `-1` combo got force-closed on KWin 6 with NVIDIA proprietary. Tearing-control async hint removed for backgrounds.
- **Wayland fractional scaling no longer produces black borders.** `xdg_output.logical_size` and `logical_position` now actually populate the output state.
- **Live-wallpaper defaults.** `pause_on_fullscreen` now defaults to `false` so the wallpaper keeps rendering behind fullscreen apps. This is what a live interactive wallpaper is supposed to do; the previous default produced a "black left monitor" symptom on multi-monitor setups when the auto-pause feature wrongly attributed a fullscreen window to the wrong output.
- **Occlusion attribution fixed.** Toplevel windows now track every output they have entered (was: a single, last-write-wins pointer), so a fullscreen window on one output never marks another output as occluded.
- **First paint is never skipped.** Layer-shell surfaces always commit an initial buffer, so wlroots and Hyprland 0.40+ don't unmap the layer when the daemon thinks an output is occluded at startup.
- **`interactive_mouse` config.** Disable shader cursor reactivity without giving up `iMouse`-using shaders. Real mouse input is unaffected.
- **Mouse reactivity scaled by `shader_speed`.** Slow shaders get a gentle camera; fast ones track the cursor. Computed around the screen centre so a still cursor maps to a still camera.
- **Cycle skips shaders that fail to compile.** Shadertoy shaders that reference a missing Common-tab helper no longer freeze the cycle on the previous shader; the daemon advances to the next one and logs a clear warning.
- **Hardened against bad shader files.** `wrap_pass_source` no longer overflows the heap on input that the Shadertoy compatibility shim expands more than 2x.
- **PID-file recycling protection.** Daemon liveness check now compares `/proc/<pid>/stat` start-time against the value recorded in the PID file. Will not SIGTERM a recycled PID.
- **ASan-clean.** Builds under `-Db_sanitize=address,undefined` with no warnings.
- **Per-pass GL hot-path tightened.** Constant uniforms (`iChannelResolution`, `iTimeDelta`, `iFrameRate`, `iSampleRate`) uploaded once at link time. `iDate` cached at 1 Hz instead of `time()`+`localtime()` per pass. `glGetIntegerv(GL_FRAMEBUFFER_BINDING)` re-queried only when the surface changes. FPS-watermark VBO created once. `glGetError` calls in the image-mode hot path gated to debug builds.

## Install

**Build from this fork:**
```bash
git clone https://github.com/obelisk-complex/neowall && cd neowall
meson setup build && ninja -C build
sudo cp build/neowall /usr/local/bin/neowall
```

**Pre-built binaries** are not yet published for this fork. See [Releases](https://github.com/obelisk-complex/neowall/releases) when they appear. Upstream `1ay1/neowall` releases are an option but predate this fork's changes.

## Config

Lives at `~/.config/neowall/config.vibe`.

**Shader wallpaper:**
```
default {
  shader retro_wave.glsl
  shader_speed 0.8
}
```

**Shader cycle (point at a directory):**
```
default {
  shader ~/.config/neowall/shaders/
  duration 30
  transition fade
  shader_speed 0.4
  interactive_mouse false
}
```

`duration 0` disables auto-cycling; advance manually with `neowall next`.

**Image slideshow:**
```
default {
  path ~/Pictures/Wallpapers/
  duration 300
  transition glitch
}
```

**Multi-monitor (Wayland with named outputs):**
```
output {
  DP-1     { shader matrix_rain.glsl }
  HDMI-A-1 { path ~/Pictures/ duration 600 }
}
```

On X11 the daemon currently exposes a single virtual-screen output named `X11 Screen`; per-monitor configuration there is a follow-up.

### Configuration keys

| Key | Default | Description |
|-----|---------|-------------|
| `path` / `shader` | — | File or directory. Cycling triggers when path is a directory of matching extensions. |
| `duration` | 0 | Seconds per item in cycle. 0 = manual. |
| `transition` | `fade` | `none`, `fade`, `slide-left`, `slide-right`, `glitch`, `pixelate` (image cycles only). |
| `transition_duration` | 300 | Milliseconds. |
| `shader_speed` | 1.0 | Multiplier on `iTime`. Also scales `iMouse` reactivity around screen centre. |
| `shader_fps` | 60 | Target frame rate. |
| `vsync` | false | Sync to monitor refresh; ignores `shader_fps` when on. |
| `pause_on_fullscreen` | **false** | Pause rendering when fully occluded by a fullscreen window. Default off in this fork. |
| `interactive_mouse` | true | Feed cursor into shader `iMouse`. Set false to lock to centre. |
| `show_fps` | false | On-screen FPS watermark. |
| `channels` | — | Optional array of `iChannel` texture paths or named defaults (`rgba_noise`, `gray_noise`, `blue_noise`, `wood`, `abstract`). |
| `mode` | `fill` | Image scaling: `center`, `stretch`, `fit`, `fill`, `tile`. |

## Commands

```bash
neowall          # start (foreground: -f -v)
neowall kill     # stop
neowall next     # next wallpaper / shader
neowall pause    # pause rendering
neowall resume   # resume
neowall list     # show cycle
neowall set 3    # jump to index 3
neowall current  # what's playing?
```

## Shaders

Drop `.glsl` files under `~/.config/neowall/shaders/`. Shadertoy shaders mostly drop in directly — write the `mainImage(out vec4 fragColor, in vec2 fragCoord)` body and the daemon supplies the standard `iTime`, `iResolution`, `iMouse`, `iFrame`, `iChannel0..3`, `iChannelResolution`, `iChannelTime`, `iDate`, `iSampleRate`, `iTimeDelta`, `iFrameRate` uniforms.

**Caveats with Shadertoy ports:**
- Shadertoy's "Common" tab lives in a separate UI tab — copy that into the same file (above `mainImage`) when porting, or you'll see `undeclared identifier 'Foo'` compile errors.
- `iChannel0..3` default to procedural noise textures. Audio FFT (`texelFetch(iChannel0, ivec2(.7,0), 0)`) returns the noise value rather than mic input; the visual differs but does not crash.
- Buffer-A / multi-pass feedback shaders are detected when `// Buffer A` markers are present; otherwise the shader runs as a single Image pass.
- `samplerCube` and compute / SSBO passes are not supported.

When a shader fails to compile during a cycle the daemon now logs a warning and advances to the next shader; rapid taps of `neowall next` no longer end up showing the same shader twice.

## How it works

```
┌────────────────────────────────────────┐
│            NeoWall Daemon              │
├────────────────────────────────────────┤
│  Config Parser → Event Loop → Shaders  │
├────────────────────────────────────────┤
│  Wayland (layer-shell)  │  X11 (EWMH)  │
├────────────────────────────────────────┤
│           EGL / OpenGL 3.3             │
└────────────────────────────────────────┘
```

- Pure C, single binary.
- GPU does the work; the CPU sleeps on `timerfd` and `signalfd`.
- One window covers the virtual screen on X11 (per-monitor on Wayland via `wlr-layer-shell`).
- The X11 backend also writes the rendered framebuffer into `_XROOTPMAP_ID` once per second, so apps using fake-transparency (Conky, transparent terminals) see live wallpaper pixels.

## Compatibility

| Environment | Status |
|-------------|--------|
| Linux X11 — Mint Cinnamon, KDE 5/6, GNOME, Xfce, MATE | Works (managed window with EWMH desktop type) |
| Linux X11 — i3, bspwm, dwm, xmonad | Works (override-redirect; was the original target) |
| Linux Wayland — sway, Hyprland, river (wlroots) | Works (layer-shell BACKGROUND) |
| Linux Wayland — KDE Plasma 6 / KWin 6 | Works (layer-shell BACKGROUND, exclusive zone 0) |
| Linux Wayland — GNOME / Mutter | Falls back to subsurface; layer-shell is not exposed by Mutter |
| Wayland fractional scaling (1.25/1.5/1.75/2.0) | Works (logical size honoured) |
| Multi-monitor X11 | Works (covers full virtual-screen bounding box) |
| Per-monitor different shaders on X11 | Not supported (single virtual-screen window) |
| Per-monitor different shaders on Wayland | Works (`output { … }` blocks) |

## Caveats

- **GNOME Wayland:** Mutter does not expose the wlr layer-shell protocol. The daemon falls back to a subsurface; behaviour is best-effort. Switch to a GNOME X11 session for full support.
- **No video wallpapers:** Use [mpvpaper](https://github.com/GhostNaN/mpvpaper) for that. Live shaders only.
- **KDE Plasma:** Desktop icons may hide. Use a dock or a desktop-icon widget.
